## 1. プロダクトビジョン

### 1.1 コンセプト
**「やったことが、勝手に日記になる — 3年分の自分と出会う」**

ENSO JOURNALは**3年日記 × 行動ログ自動生成**アプリ。
ENSO FOCUS（集中記録）・ENSO TASK（タスク管理）のデータから
今日の行動ログが自動生成され、ユーザーは一言コメントを添えるだけ。

さらに「3年日記」の仕組みで、**去年・一昨年の同じ日の自分**と出会える。
過去の自分が何をし、何を感じていたかを振り返ることで、成長を実感できる。

### 1.2 タグライン
- JA: `やったことが、勝手に日記になる`
- EN: `Your actions become your journal`

### 1.3 コアバリュー
1. **ゼロ入力**: ENSOデータから自動で日記が生成される
2. **30秒完結**: 開く → 見る → 一言添える → 終わり
3. **3年日記**: 去年・一昨年の同じ日が自動で表示される
4. **ENSO独占**: 他のジャーナルアプリでは不可能な連携

### 1.4 競合との差別化

| 競合 | やっていること | ENSO JOURNALの違い |
|------|-------------|-------------------|
| Day One | 手書き日記 + 写真 | **自動生成** + **3年日記**で過去の自分と比較 |
| Daylio | 気分トラッカー | 行動ログベース。**何をしたか**がわかる |
| 10年日記帳（紙） | 同じ日の振り返り | デジタル × 行動データ自動取得 |
| Notion日記 | 自由フォーマット | **構造化されたタイムライン** + 3年分の比較 |

**殺し文句**: 「書かなくていい3年日記。去年の今日、あなたは何をしていましたか？」

### 1.5 ターゲットユーザー
- ENSO FOCUS / TASK の既存ユーザー（最優先）
- 日記を書きたいけど続かない人
- 3年日記・5年日記に興味はあるが紙では続かない人
- 自分の成長を可視化したい人

### 3.2 3年日記セクション（コア差別化）

**去年・一昨年の同じ日**のジャーナルをカード形式で表示する。

#### 表示ルール
- 去年（1年前）の同日データがある → 表示
- 一昨年（2年前）の同日データがある → その下に表示
- データがない年 → 「1年後、ここにあなたの記録が表示されます」のプレースホルダー
- 初回利用 → 「来年の今日、あなたはこの日記を振り返ります」の導入メッセージ

#### カード表示内容
- 日付（YYYY/MM/DD）
- 気分絵文字
- 集中サマリー（FOCUS連携データがあれば）
- コメント（あれば）
- タップで詳細展開（全アクティビティログ表示）

#### 3年日記の体験設計
- **1年目**: 「来年の自分へのメッセージを残そう」感を演出
- **2年目**: 「去年の今日を振り返ろう」→ 成長実感
- **3年目**: 「2年分の自分と出会える」→ 完全体

### 3.3 自動生成アクティビティログ

#### ENSO FOCUSからの自動取得
- 完了した集中セッションを時刻付きで表示
- タグ名（FOCUSで設定したタグ）を表示
- セッション時間を表示

**データソース**: localStorage `enso-focus-sessions`

#### ENSO TASKからの自動取得（将来）
- 完了したタスクを時刻付きで表示

#### 手動エントリー
ENSOデータがなくても手動で行動ログを追加可能:
- 「+ 追加」ボタン
- 時刻 + テキスト（自由入力）
- アイコン選択（🎯集中 / ✅完了 / 📝メモ / 💡アイデア）

### 3.4 集中サマリーカード
- 今日の合計集中時間
- セッション数
- 完了タスク数（TASK連携時）
- ENSOデータがない場合: 「ENSO FOCUSで集中を記録しよう」CTA表示

### 3.5 ひとことコメント
- テキストエリア（1〜3行、最大200文字）
- プレースホルダー: 時間帯に応じて変更
  - 朝: 「今日の目標は？」
  - 昼: 「午前中はどうでした？」
  - 夜: 「今日を一言で表すなら？」
- **任意入力** — 書かなくてもジャーナルは成立する

### 3.6 気分セレクター
- 5段階の絵文字（😔😐🙂😊😄）
- **1タップで選択完了**
- 任意 — 選ばなくてもOK

### 3.7 自動保存
- コメント入力・気分選択は**リアルタイムでlocalStorageに保存**
- 保存ボタンなし
- 画面下部に「自動保存済み」の控えめな表示

## 5. データモデル

### 5.1 型定義

```typescript
// 1日分のジャーナル
interface DailyJournal {
  date: string;                  // "YYYY-MM-DD"（1日1レコード）
  mood?: 1 | 2 | 3 | 4 | 5;     // 気分（任意）
  comment?: string;              // ひとことコメント（任意、最大200文字）
  manualEntries: ManualEntry[];  // 手動エントリー
  createdAt: string;             // ISO 8601
  updatedAt: string;             // ISO 8601
}

// 手動追加の行動ログ
interface ManualEntry {
  id: string;                    // Date.now().toString()
  time: string;                  // "HH:MM"
  text: string;                  // 行動内容
  icon: "focus" | "done" | "memo" | "idea";
}
```

### 5.2 ストレージキー

| キー | 型 | 内容 |
|-----|---|------|
| `enso-journal-entries` | `DailyJournal[]` | ジャーナルエントリー（全期間） |
| `enso-journal-theme` | `ThemeMode` | テーマ設定 |
| `enso-journal-locale` | `Locale` | 言語設定 |
| `enso-journal-onboarded` | `boolean` | オンボーディング完了フラグ |

### 5.3 3年日記のデータ取得ロジック

```typescript
function getPastYearEntries(date: string): DailyJournal[] {
  const entries = storage.get<DailyJournal[]>("entries") ?? [];
  const [y, m, d] = date.split("-");
  const results: DailyJournal[] = [];

  // 1年前
  const oneYearAgo = `${Number(y) - 1}-${m}-${d}`;
  const entry1 = entries.find(e => e.date === oneYearAgo);
  if (entry1) results.push(entry1);

  // 2年前
  const twoYearsAgo = `${Number(y) - 2}-${m}-${d}`;
  const entry2 = entries.find(e => e.date === twoYearsAgo);
  if (entry2) results.push(entry2);

  return results;
}
```

### 5.4 データ容量対策
- localStorage上限: ~5MB
- 3年分の日記 ≒ 1095レコード × ~500byte = ~550KB → 余裕あり
- 画像なし（テキストのみ）なので容量問題は発生しにくい

### 5.5 ENSO連携（読み取り専用）

| 読む側 | キー | 内容 |
|--------|-----|------|
| JOURNAL → FOCUS | `enso-focus-sessions` | 今日の集中セッション |
| JOURNAL → TASK | `enso-task-*`（TBD） | 完了タスク |

**注意**: 別オリジンのため、同一オリジン化後に連携有効。
MVP段階では手動エントリー + コメント + 気分 + 3年日記で完結。

## 9. 習慣化メカニズム

### 9.1 ストリーク
- ENSOデータだけで日記が生成されるので、**FOCUSを使うだけでストリークが伸びる**
- 7日、30日、100日、365日でマイルストーン

### 9.2 3年日記の心理効果
- 「来年の自分がこれを読む」→ コメントを残すモチベーション
- 「去年の自分がこんなことをしていた」→ 成長実感 → 継続意欲
- 時間が経つほど価値が増す → **長期リテンションが自然に上がる**

### 9.3 最小摩擦設計
- **入力ゼロでもジャーナルが成立**
- コメントも気分も任意
- 「何も書かなくても、あなたの1日は記録されている」
